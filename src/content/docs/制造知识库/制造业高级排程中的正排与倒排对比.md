### 核心区别

| **维度**     | **正排 (正序排程)**             | **倒排 (倒序排程)**            |
| ------------ | ------------------------------- | ------------------------------ |
| **起点**     | 从订单接收/生产开始时间正向推算 | 从订单交付截止时间反向推算     |
| **优先级**   | 资源利用率优先                  | 交期保障优先                   |
| **时间轴**   | 当前时间 → 未来时间             | 交付截止时间 → 当前时间        |
| **产能约束** | 可能因资源不足延后交期          | 可能因资源不足需要调整生产策略 |

### 应用场景对比

正排适用场景

- 产能有限时优化资源分配
- 长周期生产项目（如飞机制造）
- 按批次生产的流程型制造
- 原料供应不稳定的情况

倒排适用场景

- 交期严格的关键订单
- JIT（准时制）生产模式
- 短周期快消品制造
- 供应链响应敏捷的环境

### 算法特性对比

| **特性** | 正排                       | 倒排                        |
| -------- | -------------------------- | --------------------------- |
| 时间计算 | 前推法(Forward Scheduling) | 后推法(Backward Scheduling) |
| 瓶颈处理 | 暴露产能缺口               | 强制满足交期                |
| 灵活性   | 更适应常规生产节奏         | 更适合紧急订单处理          |
| 缓冲设置 | 通常在工序间设置时间缓冲   | 更多在起始阶段预留安全时间  |

### 现代APS系统常采用：

1. **关键路径倒排**：对核心工序倒排
2. **非关键路径正排**：辅助工序正排
3. **动态缓冲调整**：根据实时产能变化自动优化
