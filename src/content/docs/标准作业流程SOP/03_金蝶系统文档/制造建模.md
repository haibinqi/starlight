1. 物料新增流程 工程部新增通知采购、计划、仓库、等部门维护关键属性后审核。特殊情况可以先审核但是必须通知（暂时不启用流程，流程相对比较耗时初期不建议）

2. 物料上传图片和附件

    1. 导出物料数据（导出数据按导入模板）
    2. 模板字段“picturefield”维护图片文件（123.png），图片文件夹创建压缩包
    3. 导入维护后的物料数据，系统识别到图片字段有内容会提醒用户上传资源压缩包
    4. 完成导入图片或者附件，验证
    5. 参考社区资料https://vip.kingdee.com/knowledge/791471779687983104?productLineId=40&isKnowledge=2&lang=zh-CN

    | 属性     | 核心字段                                                     | 部门责任人        |
    | -------- | ------------------------------------------------------------ | ----------------- |
    | 基础属性 | 物料编码、名称、规格型号、图号                               | 蒋颜              |
    | 采购属性 | 默认供应商、固定提前期                                       | 张晓燕 and 沈开林 |
    | 计划     | 提前期、批量规则（生产订单合并）、最小订货量、安全库存、批号序列号管理 | 计划/王子强       |
    | 仓库     | 默认仓库仓位（来料）、发料仓库仓位、入库仓库仓位             | 范燕娟            |
    | 质量     | 来料检验、生产检验、发货检验、检验方案                       | 侯水辉            |

3. 虚拟件设置 ****虚拟件PH必须设置为生产领料，系统才能自动跳层到下一级

4. 物料版本管理（暂时管理方法）

    - 工程部新建升版后的物料（控制规则：只有功能、尺寸、配合关系变更才新建新料）
    - 仓库做库存形态转换，把原有库存转移到升版后的物料
    - 原料号停用

5. BOM变更联动用料清单更新 ****工程部门变更bom后必须更新生产工单的用料清单，修改子件信息后必须反向更新bom清单\

6. 创建组织的BOM修改后，且同步更新使用组织的BOM后，后者再更新用料清单，特定条件下会导致用料清单中重复增加子项。对此同步功能进行了优化完善，避免用料清单中出现重复的数据。
